Introduction to HTML

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It provides the basic structure and content of websites by using a system of elements and tags to define different parts of a document.

HTML was first developed by Tim Berners-Lee in 1990 and has evolved through several versions. The latest version, HTML5, introduced many new features and semantic elements that make web development more efficient and accessible.

HTML is not a programming language but a markup language that uses tags to structure content. Every web page you see on the internet is built with HTML as its foundation.

A basic HTML document structure looks like this:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Welcome to HTML!</h1>
    <p>This is my first paragraph.</p>
</body>
</html>

Key concepts in HTML include:

  • Elements are the building blocks of HTML pages
  • Tags are keywords enclosed in angle brackets
  • Attributes provide additional information about elements
  • Semantic markup improves accessibility and SEO
  • Proper nesting and hierarchy are essential
HTML Elements and Tags

HTML elements are defined by tags, which are keywords enclosed in angle brackets. Most elements have an opening tag and a closing tag, with content placed between them. Understanding the proper use of different HTML elements is crucial for creating well-structured web pages.

HTML elements can be categorized into different types based on their purpose and behavior. Block-level elements take up the full width available and start on a new line, while inline elements only take up as much width as necessary.

Common HTML Elements:

<!-- Headings -->
<h1>Main Title</h1>
<h2>Section Title</h2>
<h3>Subsection Title</h3>

<!-- Text Elements -->
<p>This is a paragraph of text.</p>
<strong>Bold text</strong>
<em>Emphasized text</em>

<!-- Links and Images -->
<a href="https://example.com">Visit Example</a>
<img src="image.jpg" alt="Description">

HTML5 introduced semantic elements that provide meaning to the structure:

<header>
    <nav>
        <ul>
            <li><a href="#home">Home</a></li>
            <li><a href="#about">About</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <section>
            <h2>Article Title</h2>
            <p>Article content goes here.</p>
        </section>
    </article>
    <aside>
        <p>Sidebar content</p>
    </aside>
</main>

<footer>
    <p>© 2024 My Website</p>
</footer>

Important HTML elements include:

  • <div> and <span> for generic containers
  • <header>, <nav>, <main>, <footer> for page structure
  • <article>, <section>, <aside> for content organization
  • <form>, <input>, <button> for user interaction
  • <table>, <ul>, <ol> for structured data
CSS Fundamentals

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation and formatting of HTML documents. CSS controls the visual appearance of web pages, including colors, fonts, layouts, spacing, and responsive design.

CSS works by selecting HTML elements and applying style rules to them. The "cascading" nature means that styles can inherit from parent elements and can be overridden by more specific rules.

CSS separates content from presentation, allowing you to maintain consistent styling across multiple pages and making websites easier to maintain and update.

There are three ways to include CSS in your HTML:

<!-- 1. Inline CSS -->
<p style="color: blue; font-size: 16px;">Blue text</p>

<!-- 2. Internal CSS -->
<head>
    <style>
        p {
            color: red;
            font-family: Arial, sans-serif;
        }
    </style>
</head>

<!-- 3. External CSS -->
<head>
    <link rel="stylesheet" href="styles.css">
</head>

Basic CSS syntax consists of selectors, properties, and values:

/* CSS Rule Structure */
selector {
    property: value;
    another-property: another-value;
}

/* Example */
h1 {
    color: #333;
    font-size: 2em;
    text-align: center;
    margin-bottom: 20px;
}

Key CSS concepts include:

  • Selectors target specific HTML elements
  • Properties define what aspect of the element to style
  • Values specify how the property should be styled
  • The box model controls spacing and layout
  • Cascading and specificity determine which styles apply
CSS Selectors and Properties

CSS selectors are patterns used to target and select HTML elements for styling. Understanding different types of selectors and their specificity is essential for effective CSS development and avoiding styling conflicts.

CSS provides various types of selectors, each with different levels of specificity. Learning when and how to use each selector type will help you write more maintainable and efficient stylesheets.

Common CSS Selectors:

/* Element Selector */
p { color: black; }

/* Class Selector */
.highlight { background-color: yellow; }

/* ID Selector */
#header { font-size: 24px; }

/* Descendant Selector */
nav ul li { display: inline-block; }

/* Child Selector */
article > p { margin-bottom: 15px; }

/* Pseudo-classes */
a:hover { color: red; }
input:focus { border-color: blue; }

/* Attribute Selectors */
input[type="email"] { width: 100%; }
a[href^="https"] { color: green; }

Common CSS properties for styling and layout:

/* Typography */
.text-styles {
    font-family: 'Arial', sans-serif;
    font-size: 16px;
    font-weight: bold;
    line-height: 1.6;
    text-align: center;
    text-decoration: underline;
}

/* Colors and Backgrounds */
.colors {
    color: #333333;
    background-color: #f0f0f0;
    background-image: url('pattern.png');
    opacity: 0.8;
}

/* Box Model */
.box-model {
    width: 300px;
    height: 200px;
    padding: 20px;
    margin: 10px auto;
    border: 2px solid #ccc;
    border-radius: 8px;
}

/* Layout */
.layout {
    display: flex;
    justify-content: center;
    align-items: center;
    position: relative;
    float: left;
}

Essential CSS properties categories:

  • Typography: font-family, font-size, font-weight, line-height
  • Colors: color, background-color, opacity
  • Box Model: width, height, padding, margin, border
  • Layout: display, position, float, flexbox, grid
  • Visual Effects: box-shadow, border-radius, transform, transition
Responsive Web Design

Responsive web design is an approach that ensures web pages render well on various devices and screen sizes. With the proliferation of mobile devices, creating responsive websites has become essential for providing good user experiences across all platforms.

Responsive design uses flexible layouts, images, and CSS media queries to adapt the presentation of content based on the device's characteristics, primarily screen width, height, and orientation.

The mobile-first approach is now considered best practice, where you design for mobile devices first and then enhance for larger screens using media queries.

Key responsive design techniques include:

/* Flexible Grid System */
.container {
    max-width: 1200px;
    margin: 0 auto;
    padding: 0 20px;
}

.row {
    display: flex;
    flex-wrap: wrap;
    margin: 0 -15px;
}

.col {
    flex: 1;
    padding: 0 15px;
    min-width: 250px;
}

/* Responsive Images */
img {
    max-width: 100%;
    height: auto;
}

/* Media Queries */
@media screen and (max-width: 768px) {
    .container {
        padding: 0 10px;
    }
    
    .col {
        flex: 100%;
        margin-bottom: 20px;
    }
    
    h1 {
        font-size: 1.8rem;
    }
}

@media screen and (max-width: 480px) {
    body {
        font-size: 14px;
    }
    
    .nav-menu {
        display: none;
    }
    
    .mobile-menu {
        display: block;
    }
}

Modern CSS layout methods for responsive design:

  • Flexbox for one-dimensional layouts and component alignment
  • CSS Grid for two-dimensional layouts and complex designs
  • Media queries for device-specific styling breakpoints
  • Relative units like percentages, em, rem, and viewport units
  • Mobile-first methodology for progressive enhancement
Best Practices

Following best practices in HTML and CSS development ensures your code is maintainable, accessible, and performs well across different browsers and devices. These practices have evolved from years of web development experience and industry standards.

Good HTML and CSS practices not only improve code quality but also enhance SEO, accessibility, and user experience. They make collaboration easier and reduce maintenance costs over time.

HTML Best Practices:

<!-- Use semantic HTML5 elements -->
<article>
    <header>
        <h1>Article Title</h1>
        <time datetime="2024-01-15">January 15, 2024</time>
    </header>
    <p>Article content with proper structure.</p>
</article>

<!-- Always include alt text for images -->
<img src="chart.png" alt="Sales data showing 25% growth">

<!-- Use proper form labels -->
<label for="email">Email Address:</label>
<input type="email" id="email" name="email" required>

CSS organization and naming conventions:

/* Use meaningful class names */
.navigation-menu { }
.article-content { }
.sidebar-widget { }

/* Group related styles */
/* ==========================================================================
   Navigation Styles
   ========================================================================== */

.nav {
    display: flex;
    justify-content: space-between;
}

.nav__item {
    padding: 10px 15px;
}

.nav__link {
    text-decoration: none;
    color: inherit;
}

.nav__link--active {
    font-weight: bold;
    color: #e74c3c;
}

Essential best practices include:

  • Write semantic HTML that describes content meaning, not appearance
  • Use external CSS files to separate content from presentation
  • Follow consistent naming conventions like BEM or SMACSS
  • Optimize images and use appropriate file formats
  • Test across different browsers and devices regularly
  • Validate your HTML and CSS code for errors
  • Include proper meta tags for SEO and social sharing
  • Ensure accessibility with proper ARIA attributes and keyboard navigation
  • Use version control systems like Git for code management
  • Minify and compress files for production deployment
